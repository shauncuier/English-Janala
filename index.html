Frequently Asked Questions

1. The difference between code var, let, and const

var: Function-scoped, can be redeclared/updated, hoisted

let: Block-scoped, can be updated but not redeclared, not hoisted

const: Block-scoped, cannot be updated or redeclared, not hoisted

2. The difference between map(), forEach(), and filter()

map(): Creates new array by transforming each element

forEach(): Executes function on each element but doesn't return anything

filter(): Creates new array with elements that pass a test condition

3. Explain arrow functions and how they are different from regular functions
Arrow functions are shorter syntax for writing functions. Key differences: 1. No 'this' binding 2. Can't be used as constructors 3. Don't have arguments object 4. Can't use 'yield' within them
4. How JavaScript Promises work?
Promises are objects representing eventual completion of async operations. They have three states: 1. Pending: Initial state 2. Fulfilled: Operation completed successfully 3. Rejected: Operation failed Use .then() for success and .catch() for errors.
5. How closures work in JavaScript?
Closures are functions that can access variables from their outer scope even after the outer function has returned. They retain access to variables in their lexical scope and are commonly used for data privacy and maintaining state.
